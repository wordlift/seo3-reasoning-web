About These Samples

These HTML pages use real entity data from the production WordLift knowledge graphs used in the research. Each entity page augments the original structured data with the enhanced entity page format used in experimental conditions C3 and C6 from "Structured Linked Data as a Memory Layer for Agent-Orchestrated Retrieval".

The enhanced format adds:

  • Schema.org type breadcrumbs showing entity hierarchy
  • Embedded JSON-LD matching the production knowledge graph data
  • Linked entity navigation with dereferenceable URIs and content negotiation
  • Agent instructions (llms.txt style) with public search endpoints for KG traversal
  • Canonical links to the original schema:url to avoid confusing traditional search engines

Technical Notes

  • Real data: All entity URIs, properties, and linked entities come from the production knowledge graphs used in the experiment
  • Canonical links: Each page includes <link rel="canonical"> pointing to the entity's schema:url to avoid confusing traditional search engines
  • Public search API: Agent instructions reference a /search endpoint (no API key required) for KG traversal by external agents
  • Content negotiation: All entity URIs support Accept: application/ld+json for machine-readable formats
